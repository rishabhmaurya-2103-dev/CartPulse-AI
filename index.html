◈ Model Documentation

How CartSense
works

CartSense uses a trained Random Forest Classifier to predict whether a cart event will lead to abandonment or purchase — in real time, using your actual .pkl model files.

Data Pipeline

1
Raw e-commerce events (Oct–Nov 2019)
→
2
Filter cart events · join session features
→
3
Label encode brand & category
→
4
RandomForest · 100 trees · depth 12
→
5
Predict abandon probability
🌲

Random Forest Classifier

100 decision trees with max depth 12, balanced class weights to handle the slight imbalance between abandoned (6,714) and purchased (8,244) samples. Trained on 80% / evaluated on 20% stratified split.

🔢

Label Encoders

The exact brand_encoder.pkl and category_encoder.pkl from training are loaded at startup. 176 brands and 69 categories are faithfully encoded to match the training distribution.

⚡

Real-time Inference

Every prediction is a live call to the Flask backend, which loads the model once at startup and keeps it in memory. Typical inference time is under 5 ms per request.

📊

Dataset

Kaggle dataset "ecommerce-behavior-data-from-multi-category-store" — 1 million rows sampled from Nov 2019. Cart events were extracted and session-level features aggregated to form the training set of ~15k records.
